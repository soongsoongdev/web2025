Chapter 1 / 웹 구조 및 JavaScript 개요

1. 웹 구성 요소

  • 클라이언트 / 서버 구조
  • 브라우저와 웹 서버의 상호작용
  • 웹의 3요소: HTML, CSS, JavaScript
  • HTTP: 웹 서비스 요청과 응답을 위한 프로토콜

    - 프로토콜: 통신 규약 (절차와 메시지 형식)
    - HTTP는 하이퍼텍스트 데이터 서비스를 위한 프로토콜로, 애플리케이션 간의 연결을 맺고 데이터 교환 후 종료함
    - 메시지는 Header와 Body로 구성됨
    └ Header: 요청 유형(GET, POST 등), URL, 쿠키 등
    └ Body: HTML 문서, 파일 등 실제 데이터

  • URL: 요청 서비스 주소

    - Scheme: 접근 방식 (http, ftp, mail 등)
    - Host: 서버 도메인 이름 또는 IP 주소
    - Port: 서비스 포트 번호 (기본 80은 생략 가능)
    - Path: 서버 내의 자원 위치

  • HTML: 정보 페이지의 구조

    - 정보와 태그로 구성
    - 태그는 구조와 의미를 표현
    - CSS: 스타일 정의
    - JavaScript: 동적인 제어 및 사용자 상호작용 처리

3. JavaScript 개요

  • 자바스크립트의 특징

    - 모든 웹 브라우저에서 작동
    - 실행 결과를 바로 확인할 수 있음
    - 웹 프론트엔드, 백엔드, 앱, 서버 등 다양한 분야에 사용 가능
    - 공개 API, 라이브러리, 프레임워크 다양
    - 다이나믹한 사용자 경험 제공

  • 자바스크립트와 자바의 차이

    - 자바스크립트는 본래 '라이브스크립트'였으나 마케팅 목적상 자바스크립트로 명명됨
    - 두 언어는 전혀 다른 언어 체계를 가짐

  • ECMAScript와의 관계

    - 자바스크립트는 ECMA 국제기구에 의해 표준화됨
    - ECMA-262 문서에 정의된 스크립트 언어 사양이 'ECMAScript'
    - 자바스크립트는 ECMAScript를 따르는 대표 언어

  • ES6 (ECMAScript 2015) 이후의 변화

    - 1997년부터 ECMAScript 표준 시작
    - 2015년 ES6가 등장하면서 언어 기능 대폭 강화 (클래스, 모듈, 화살표 함수 등)
    - 이후 매년 ES6+ 버전으로 업데이트 진행 중
